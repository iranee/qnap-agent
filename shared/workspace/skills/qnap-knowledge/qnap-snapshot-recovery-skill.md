# QNAP 快照恢复实战指南

> 适用范围：QTS 5.x 快照管理与数据恢复
> 适用场景：大规模数据恢复、存储池只读模式修复、快照空间不足问题

---

## 一、快照恢复的核心空间限制

### 1.1 厚卷快照恢复的隐性限制

**关键原理（很多用户不知道）：**

快照文件保存在存储池内部，占用的是存储池的实际磁盘空间，但这些空间在 QTS 存储界面中**不可见**。即使显示"可用空间充足"，也可能无法完成恢复。

```
示例场景：
- 厚卷总容量：10T
- 显示可用空间：7T
- 待恢复快照数据：3T
- 结果：恢复失败！

原因：快照本身占用了部分空间，
厚卷可用的"实际净空间"< 显示的可用空间。
恢复到本身存储池时，数据要写回的空间已被快照占用。
```

**结论：** 厚卷快照无法恢复到本身所在的存储池，必须恢复到其他存储池或其他磁盘。

### 1.2 存储池进入只读模式

当快照数据写满厚卷可用空间时，存储池会自动切换为**只读模式**：
- 可以读取文件，但无法写入任何内容
- 通过 `df -h` 看起来仍有可用空间，但写入会失败

## 二、跨存储池恢复流程

### 2.1 判断是否需要跨存储池恢复

```sh
# 查看存储池状态
df -h
/usr/sbin/lvm lvs 2>/dev/null
/usr/sbin/lvm vgs 2>/dev/null

# 查看快照占用
/usr/sbin/lvm lvs 2>/dev/null | grep snap

# 查看 RAID 状态
cat /proc/mdstat
```

**判断标准：**
- 目标存储池剩余空间 < 待恢复数据大小 → 需要跨存储池恢复
- 存储池处于只读模式 → 必须跨存储池恢复

### 2.2 跨存储池数据迁移步骤

```sh
# 1. 确认目标存储池（CACHEDEV2_DATA）有足够空间
df -h /share/CACHEDEV2_DATA/
du -sh /share/CACHEDEV1_DATA/<目标目录>/ 2>/dev/null

# 2. 先删除目标存储池上的快照，防止写入时撑爆目标池
# （通过 QTS GUI：存储与快照 → 快照 → 删除旧快照）

# 3. 从源存储池快照访问目录复制到目标存储池
# 快照内容通过快照访问点访问
ls /share/CACHEDEV1_DATA/.snapshot/ 2>/dev/null
ls /share/<共享名>/.snapshot/ 2>/dev/null

# 使用 cp 命令迁移（注意保留权限）
cp -a /share/<共享名>/.snapshot/<快照名>/<子目录>/ \
       /share/CACHEDEV2_DATA/<目标目录>/

# 或使用 rsync（可断点续传，更安全）
rsync -av --progress \
       /share/<共享名>/.snapshot/<快照名>/<子目录>/ \
       /share/CACHEDEV2_DATA/<目标目录>/
```

### 2.3 删除快照释放只读限制

数据迁移完成后，需要删除源存储池的所有快照才能解除只读模式：

**通过 QTS GUI（推荐）：**
1. 存储与快照 → 快照 → 快照库
2. 选择对应共享文件夹
3. 删除所有快照
4. 等待操作完成

**注意：** 删除快照是不可逆操作，必须确认数据已迁移完成后再执行。

### 2.4 重新挂载存储池（解除只读模式）

删除快照后，通过以下步骤修复只读存储池：

```sh
# 1. 重新初始化 LVM（重新挂载存储池）
/etc/init.d/init_lvm.sh

# 2. 等待挂载完成（通常 10-30 秒）

# 3. 验证挂载状态
df -h | grep share
mount | grep CACHEDEV
```

### 2.5 文件系统检查（最关键的一步）

存储池重新挂载后，需要通过 QTS GUI 进行文件系统检查：

```
【存储与快照总管】
  → 右击存储池下面的磁盘名（厚卷/精简卷）
  → 管理
  → 操作
  → 检查文件系统
```

**等待检查完成后，只读属性将自动消除。**

## 三、厚卷转精简卷（预防再次出现此问题）

### 3.1 为什么建议转换

厚卷开启快照后：
- 快照会不断占用存储池空间
- 可用空间看起来充足，实际净空间不断减少
- 恢复大量数据时容易再次触发只读模式

精简卷（Thin Volume）：
- 空间按需分配，不预先占用
- 快照空间管理更灵活
- 恢复数据时空间限制更少

### 3.2 转换方式

**通过 QTS GUI 转换（快速，不丢数据）：**
```
存储与快照 → 存储 → 存储池
  → 右击对应卷
  → 转换为精简卷
  → 确认转换
```

**注意：**
- 厚卷转精简卷速度很快（通常几分钟）
- 转换期间可以正常使用，不影响读写
- 转换后原有数据完整保留
- 重要：转换前确认有足够的存储池可用空间

## 四、实战总结：大规模数据恢复检查清单

恢复前必须确认：

```
[ ] 目标存储池可用空间 > 待恢复数据大小 × 1.5（预留余量）
[ ] 目标存储池上的旧快照已清理（防止写满）
[ ] 已准备备用存储路径（如有另一个存储池）
[ ] 不要尝试将大量数据恢复到快照所在的同一存储池
[ ] 恢复完成后通过 QTS GUI 运行文件系统检查
```

恢复后检查：
```
[ ] df -h 确认可用空间符合预期
[ ] 随机抽查几个文件确认内容完整
[ ] 确认只读模式已解除（尝试创建一个测试文件）
[ ] 建议立即将厚卷转为精简卷，避免再次出现此问题
```

## 五、安全要求

- 查询快照状态：只读，无需确认
- 删除快照：必须确认（不可逆，删除后无法恢复到该时间点）
- 跨存储池数据迁移：必须确认（操作耗时且影响 NAS 性能）
- 文件系统检查：建议通过 GUI 操作（更安全可靠）
- 只读存储池修复：必须确认，涉及存储层操作
